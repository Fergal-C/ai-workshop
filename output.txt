#36327 [bug] [actionable] — [compiler Bug]: AST contract violation: babel-plugin-react-compiler writes `Symbol()` into Node.loc, breaks downstream serialization (jest-worker / v8.serialize / IPC)
  The React compiler uses a `Symbol()` sentinel for missing source locations and writes it into AST `Node.loc` fields, violating the `babel` `Node.loc: sourceLocation | null` contract and causing `v8.serialize` / `child_process.send` to throw `Symbol() could not be cloned`. A minimal standalone reproduction repository is provided demonstrating the failure.

#36321 [bug] [actionable] — fix false hydration mismatch when client renders portal that returned null on server

#36318 [bug] [actionable] — Bug: useOptimistic fails to revert when independent async actions overlap due to global scope entanglement
  The `useOptimistic` revert mechanism is gated on a global per-root pending action count in `ReactFiberAsyncAction.js`, so a slow async action in one component prevents an unrelated component's optimistic UI from reverting even after its own action completes. A CodeSandbox reproduction is provided demonstrating the cross-component entanglement issue.

#36317 [bug] [actionable] — [verified] fix(reconciler): recheck store snapshot after subscribe
  When `useSyncExternalStore` is used with `suspense` and `StrictMode`, a remount path skips the post-subscribe snapshot re-check, which can cause the hook to miss a store mutation that occurs during `subscribe()`. The PR fixes this by re-checking the snapshot immediately after `subscribe(handleStoreChange)` returns and references a related issue (#35550).
